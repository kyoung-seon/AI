# 실습 — 코랩 전이학습으로 나만의 AI를 만들어 포니봇 주행 제어 🧠🚗

> **AI 기반 자율로봇 시스템 구현 과정**
> Teachable Machine이 "자동"으로 해주던 학습 과정을 이번엔 **코드로 직접** 해봅니다.
> **데이터 수집 → 전이학습(MobileNetV2) → TF.js 변환 → 웹 배포 → 로봇 제어**,
> 실제 AI 개발자가 거치는 전 과정을 그대로 체험합니다.

---

## 0. 전체 그림

```
[구글 코랩]                        [GitHub Pages]                [포니봇]
 사진 수집 → MobileNetV2 전이학습    tfjs_model/ + 웹앱(HTTPS)     pony_ble_drive.py
 → TF.js 변환 → tfjs_model.zip  →   카메라 분류(TF.js)            on_rx → motor.drive()
                                    → Web Bluetooth ──────────▶  forward/backward/left/right/stop
```

- **Teachable Machine과 무엇이 다른가요?**
  같은 원리(전이학습)지만, 이번엔 모델 구조·학습률·에포크를 **우리가 직접 코드로** 결정합니다.
  → 정확도가 낮을 때 "왜?"를 분석하고 고칠 수 있게 됩니다.
- 분류는 **폰/PC 브라우저 안에서** 돌아갑니다 (서버·인터넷 분류 요청 불필요).
- 보내는 명령은 기존 무선 AI 실습과 완전히 동일 → **같은 수신 펌웨어** 재사용.

## 1. 데이터 준비 — 표지판 카드 만들기 (10분)

1. A4 용지 4장에 화살표를 **크고 진하게** 그립니다: `↑`(전진) `↓`(후진) `←`(좌회전) `→`(우회전)
2. `stop`은 카드가 필요 없습니다 — **아무것도 없는 배경**이 곧 정지 신호입니다.

> 💡 **왜 배경을 stop으로 하나요?**
> 분류 모델은 항상 5개 중 하나를 답해야 합니다. "카드 없음" 상황을 학습시키지 않으면
> 빈 화면을 보고도 엉뚱한 방향을 출력해 로봇이 폭주합니다. 배경=stop은 **안전장치**입니다.

## 2. 코랩에서 전이학습 (30분)

### 2-1. 노트북 열기

1. [colab.research.google.com](https://colab.research.google.com) 접속 → **업로드** 탭 → `colab_transfer_learning.ipynb` 선택
2. 메뉴 **런타임 → 런타임 유형 변경 → T4 GPU** 선택 (학습 속도 ↑)

### 2-2. 데이터 수집 (노트북 2단계)

- **방법 A (권장)**: 폰으로 클래스당 40~50장 촬영 → 폴더 정리 → `dataset.zip` 업로드
- **방법 B**: 코랩 웹캠 촬영 셀로 그 자리에서 수집

```
dataset/
 ├─ forward/  ├─ backward/  ├─ left/  ├─ right/  └─ stop/
```

⚠️ **폴더 이름 = 로봇 명령**입니다. 철자 하나라도 다르면 로봇이 움직이지 않습니다.

> 📸 좋은 데이터 체크리스트
> - [ ] 클래스당 30장 이상 (권장 50장)
> - [ ] 거리·각도·손 위치를 조금씩 바꿔가며 촬영
> - [ ] 실제 시연할 장소와 비슷한 배경·조명
> - [ ] `stop`은 카드 없는 배경만 (사람 손이 나와도 OK — 실제 상황과 같게)

### 2-3. 학습 → 그래프 확인 (노트북 3~5단계)

셀을 순서대로 실행합니다. 여기서 **꼭 확인할 것**:

| 확인 포인트 | 기준 | 문제 시 조치 |
|---|---|---|
| `클래스 순서` 출력 | `['backward','forward','left','right','stop']` | 폴더 이름 오타 수정 |
| 학습 곡선 `val_accuracy` | **0.90 이상** | 데이터 추가, 미세조정 셀 실행 |
| train↑ 인데 val↓ (과적합) | 두 곡선이 벌어지지 않아야 함 | 사진 다양성 늘리기 |
| 예측 테스트 이미지 | 초록 제목(정답)이 대부분 | 빨간(오답) 클래스 사진 보강 |

### 2-4. TF.js 변환 → 다운로드 (노트북 6단계)

마지막 셀까지 실행하면 `tfjs_model.zip`이 다운로드됩니다. 압축을 풀면:

```
tfjs_model/
 ├─ model.json                  ← 모델 구조
 ├─ group1-shard1of3.bin ...    ← 학습된 가중치 (여러 개)
 └─ labels.json                 ← 클래스 이름 (웹앱이 읽음)
```

## 3. GitHub Pages 배포 (10분)

1. GitHub에서 새 저장소 생성 (예: `my-ai-drive`, **Public**)
2. **Add file → Upload files**로 아래를 업로드
   - `custom_model_drive.html`
   - `tfjs_model` **폴더째** (폴더 드래그하면 구조가 유지됩니다)
3. **Settings → Pages → Branch: `main` / `(root)` → Save**
4. 1~2분 뒤 `https://아이디.github.io/my-ai-drive/custom_model_drive.html` 접속

> ⚠️ HTTPS가 아니면 카메라·블루투스 권한이 **거부**됩니다. `content://`, 파일 직접 열기 ❌
> ⚠️ `github.com/.../blob/...` 주소는 소스 보기일 뿐 앱이 실행되지 않습니다. 반드시 `github.io` 주소로!

## 4. 포니봇 준비 (5분)

1. Thonny로 보드에 업로드: `ai_ponybot.py`(+`microbit.py`), `ble_library.py`, `pony_ble_drive.py`
2. `pony_ble_drive.py` 실행 → OLED에 `BLE WAITING` / 셸에 `포니봇 BLE 대기 중...`

> 🔧 파일을 교체했는데 옛날 동작을 하면 → **Ctrl-D 소프트 리부트** (모듈 캐시 삭제)

## 5. 실행! (10분)

웹앱에서 순서대로:

1. **📦 모델 불러오기** — `로딩 완료 (5개 클래스)` 확인
2. **🔵 블루투스 연결** — `ESP-000` 선택 (안 보이면: 안드로이드 위치(GPS) ON)
3. **📷 카메라 시작** — 클래스 막대가 실시간으로 움직이는지 확인
4. 막대만 보고 **먼저 인식 테스트** — 카드마다 초록 막대가 정확히 반응하는지
5. **▶️ 자동 전송 시작** — 이제 카드를 보여주면 포니봇이 달립니다!
6. 카드를 치우면 `stop` → 정지. 급하면 **🛑 긴급 정지**.

> 🛡️ **안전 수칙**: 처음엔 신뢰도 기준을 85% 이상으로 올리고, 로봇 주변 1m를 비우세요.
> 블루투스가 끊기면 펌웨어가 자동으로 정지시킵니다.

## 6. 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| 모델 로딩 실패 | 경로 오류 / 폴더 미업로드 | `tfjs_model/model.json`이 웹앱과 같은 저장소에 있는지, F12 콘솔에서 404 확인 |
| `labels.json` 404 | labels.json 누락 | 코랩 6단계 셀 재실행 → 재업로드 |
| 인식은 되는데 로봇이 안 움직임 | 자동 전송 꺼짐 / BLE 미연결 / 신뢰도 미달 | 전송 상태 `전송 중` 확인, 기준 낮춰보기 |
| 특정 카드만 오인식 | 해당 클래스 데이터 부족 | 그 카드 사진 20장 추가 → 재학습 |
| 교실에선 잘 되는데 시연장에선 실패 | 배경·조명이 학습 데이터와 다름 | 시연 장소에서 데이터 추가 수집 (도메인 차이 체험!) |
| 앞뒤가 반대로 감 | 모터 배선 방향 | 펌웨어 `INVERT_FB = True` |
| 코랩 설치 셀에서 빨간 `ERROR: pip's dependency resolver...` | tensorflowjs가 TF를 함께 설치하며 생기는 정상 경고 | 무시하고 진행. 단, 이후 앞쪽 학습 셀 재실행 금지 (재학습 시 런타임 재시작) |
| 변환 시 `AssignVariableOp ... random_flip` / `Identity is not in graph` | 증강 층이 export에 포함됨 | 최신 노트북의 6단계 셀 사용 (증강 제외한 추론 모델로 export) |
| `tensorflowjs_converter: command not found` | 설치 실패 또는 런타임 재시작됨 | 설치 셀 재실행 → `--version` 확인 |
| `ImportError: ble_library` | 라이브러리 미업로드 | `ble_library.py`를 보드 루트(`/`)에 업로드 |
| 폰에서 카메라/BT 거부 | HTTPS 아님 / 권한 | `github.io` 주소 확인, 자물쇠 아이콘 + OS 설정에서 권한 허용 |
| BLE 기기 안 보임 | 안드로이드 위치 꺼짐 | 위치(GPS) 켜기, iOS Safari는 미지원(Bluefy 사용) |

## 8. 도전과제

### 🥉 도전 A — 클래스 추가
`spin`(제자리 회전) 클래스를 추가해 6-클래스 모델을 만들어 보세요.
- 새 카드 제작 → 사진 수집 → 재학습 → 재배포
- 펌웨어 `on_rx`에 `spin` 처리 추가: `motor.mecanum(6, 50)` 등 활용
- 힌트: 웹앱은 `labels.json`을 읽으므로 **수정 불필요** — 펌웨어만 고치면 됩니다!

### 🥈 도전 B — 속도까지 제어
같은 카드라도 **크게 보이면(가까우면) 빠르게, 작게 보이면 느리게** 달리게 해보세요.
- 방법 1(쉬움): `forward_fast` / `forward_slow` 두 클래스로 학습
- 방법 2(어려움): 웹앱에서 명령을 `forward:80` 형식으로 보내고 펌웨어에서 `split(':')` 파싱

### 🥇 도전 C — 나만의 주제로 재학습
화살표 대신 **완전히 다른 데이터**로 로봇을 제어해 보세요.
- 예: 가위바위보(가위=left, 바위=stop, 보=forward), 학용품 분류, 재활용 쓰레기 분류
- 폴더 이름만 명령 규칙에 맞추면 코드는 한 줄도 바꿀 필요가 없습니다 — 이것이 **파이프라인의 힘**!
