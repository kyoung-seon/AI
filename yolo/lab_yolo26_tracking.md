# 실습 — YOLO26 객체 탐지로 '물체를 따라다니는 포니봇' 만들기 🎯🐕

> **AI 기반 자율로봇 시스템 구현 과정**
> 지난 실습(전이학습)이 **분류** — "화면에 무엇이 있나?" 였다면,
> 이번 실습은 **객체 탐지** — "무엇이 **어디에, 얼마나 크게** 있나?" 입니다.
> 위치와 크기 정보가 생기는 순간, 로봇은 단순 반응을 넘어 **물체를 추적**할 수 있게 됩니다.

---

## 0. 전체 그림

```
[구글 코랩]                    [GitHub Pages]                    [포니봇]
 YOLO26(사전학습) 체험           yolo26n.onnx + 웹앱(HTTPS)         pony_ble_drive.py (지난 실습 재사용!)
 → ONNX 내보내기(320px)     →    ONNX Runtime Web 실시간 탐지   →   forward/backward/left/right/stop
 → 출력 (1,300,6) 검증           위치→조향 · 크기→거리 제어  ──BLE──▶
```

**추적 제어 규칙** (웹앱에 구현되어 있음 — 학생이 읽고 이해할 핵심 로직):

| 조건 | 판단 | 명령 |
|---|---|---|
| 물체 중심이 화면 왼쪽 (cx < 0.38) | 왼쪽에 있다 | `left` |
| 물체 중심이 화면 오른쪽 (cx > 0.62) | 오른쪽에 있다 | `right` |
| 중앙 & 박스가 목표보다 작음 | 멀리 있다 | `forward` (다가감) |
| 중앙 & 박스가 목표보다 큼 | 너무 가깝다 | `backward` (물러남) |
| 중앙 & 적정 크기 | 딱 좋은 거리 | `stop` |
| 4프레임 연속 미탐지 | 놓쳤다 | `stop` (안전) |

> 💡 이것이 로봇공학의 고전 기법 **시각 서보잉(Visual Servoing)**의 가장 단순한 형태입니다.

## 지난 실습과의 차이 — 왜 배우는가

| | 전이학습 (지난) | YOLO26 (이번) |
|---|---|---|
| AI 과제 | 분류 | 객체 탐지 |
| 학습 | 우리가 직접 (사진 수집→학습) | **불필요** — COCO 80종 사전학습 그대로 |
| 브라우저 런타임 | TensorFlow.js | **ONNX Runtime Web** |
| 후처리 | argmax 한 줄 | YOLO26 덕분에 역시 간단! (아래 참고) |
| 제어 정보 | 클래스 1개 | 클래스 + **위치 + 크기** |

**YOLO26의 교육 포인트**: 예전 YOLO는 수천 개 후보 박스를 NMS(비최대 억제)로 걸러야 해서
브라우저 구현이 복잡했습니다. YOLO26은 **end-to-end(NMS-free)** 구조라 모델 출력이
곧바로 최종 결과 `(1, 300, 6)` = [x1,y1,x2,y2,신뢰도,클래스]×300입니다.
→ 자바스크립트에서 "신뢰도 높은 행만 골라 쓰기"로 끝. 최신 모델일수록 **배포가 쉬워지는** 흐름을 체험합니다.

## 1. 코랩 — YOLO26 체험 & ONNX 내보내기 (20분)

`yolo26_colab_export.ipynb`를 코랩에 업로드하고 순서대로 실행합니다.

| 단계 | 하는 일 | 확인 포인트 |
|---|---|---|
| 1 | ultralytics 설치, `yolo26n.pt` 로딩 | 클래스 80종 출력 |
| 2 | 샘플/내 사진으로 탐지 | 박스가 그려진 이미지 |
| 3 | 출력 구조 설명 (읽기) | `(1,300,6)`의 의미 이해 |
| 4 | `model.export(format="onnx", imgsz=320)` | `yolo26n.onnx` 생성 (~10MB) |
| 5 | **onnxruntime으로 검증** | `출력 형태 (1,300,6)` + `✅ 검증 통과` |
| 6 | 다운로드 | `yolo26n.onnx` 확보 |

> ⚠️ **imgsz=320을 바꾸지 마세요.** 웹앱의 `IMG=320` 상수와 일치해야 합니다.
> (더 정확하게 하고 싶으면 둘 다 640으로 — 대신 브라우저에서 3~4배 느려집니다)
> 📌 이번엔 학습이 없으므로 GPU 런타임 없이도 됩니다.

## 2. GitHub Pages 배포 (10분)

1. 저장소에 `yolo_tracking_drive.html` + `yolo26n.onnx` 업로드 (같은 위치)
2. Settings → Pages 활성화 → `https://아이디.github.io/저장소/yolo_tracking_drive.html` 접속

> 💡 지난 실습 저장소에 그대로 추가해도 됩니다 — 파일 2개만 얹으면 끝.
> ⚠️ onnx 파일이 10MB라 첫 로딩에 몇 초 걸립니다. GitHub 파일 업로드 한도(25MB 웹/100MB git) 안이라 문제없습니다.

## 3. 포니봇 준비 (2분)

**지난 실습과 완전히 동일합니다.** `pony_ble_drive.py` 실행 → `BLE WAITING`.
같은 5개 명령(forward/backward/left/right/stop)을 쓰기 때문에 펌웨어 수정이 전혀 없습니다 —
**웹앱(두뇌)만 갈아끼우면 로봇(몸)은 그대로**라는 파이프라인 설계의 장점을 확인하세요.

## 4. 실행! (15분)

1. **📦 모델 불러오기** → `로딩 완료`
2. **📷 카메라 시작** → 화면에 박스가 그려지는지 확인 (대상 클래스=초록, 나머지=파랑)
3. 드롭다운에서 추적할 물체 선택 (기본 person)
4. **가로 위치/크기 수치가 실시간으로 변하는지** 관찰 — 제어 로직이 보는 값입니다
5. **🔵 블루투스 연결** → **▶️ 자동 전송 시작**
6. 물체를 좌우로 움직이면 로봇이 방향을 틀고, 뒤로 물러나면 따라옵니다!

**튜닝 가이드**
- 로봇이 자꾸 헛도는 경우 → 신뢰도 기준 ↑ (60~70%)
- 로봇이 너무 바짝 붙는 경우 → 목표 크기 ↓
- 조향이 반대인 경우 → **좌우 반전** 체크 (로봇이 카메라를 마주보는 배치일 때 필요)
- 사람이 많은 교실에서 person 추적 → 다른 사람을 잡을 수 있음! `teddy bear` 같은 고유 물체 추천

> 🛡️ **안전**: 물체를 화면 밖으로 치우면 4프레임 뒤 자동 정지합니다. 급하면 🛑 긴급 정지.

## 5. 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| 모델 로딩 실패 (404) | onnx 미업로드/경로 오류 | 웹앱과 같은 폴더에 `yolo26n.onnx` 확인, F12 콘솔 |
| 로딩은 되는데 박스가 안 그려짐 | 신뢰도 기준이 너무 높음 / 조명 어두움 | 기준 30%로 낮춰 테스트, 조명 확보 |
| 박스 위치가 이상함 | 코랩 imgsz ≠ 웹앱 IMG | 둘 다 320인지 확인 후 재내보내기 |
| 추론 속도 700ms+ | 저사양 CPU | 다른 탭 닫기, Chrome 최신 버전(WebGPU 지원 시 자동 가속) |
| 탐지는 되는데 로봇 무반응 | 전송 일시정지 / BLE 미연결 | `전송 중` 확인 |
| 로봇이 좌우를 반대로 돎 | 로봇-카메라 배치 문제 | **좌우 반전** 체크 |
| 계속 forward만 보냄 | 목표 크기가 너무 큼 | 목표 크기를 현재 크기 수치 근처로 조정 |
| 엉뚱한 물체를 따라감 | 비슷한 물체 다수 | 신뢰도 ↑, 고유한 물체로 변경 |

## 7. 도전과제

### 🥉 도전 A — 대기 모드 업그레이드
지금은 물체를 놓치면 그냥 멈춥니다. **마지막으로 봤던 방향으로 제자리 회전하며 찾기**를 추가해 보세요.
- 힌트: `decide()` 함수에서 `miss` 카운터가 늘어날 때, 마지막 `best.cx`를 기억해 두었다가 left/right 전송

### 🥈 도전 B — 부드러운 제어 (비례 제어 맛보기)
좌/우/직진의 3단계 대신, 중심에서 벗어난 정도에 **비례**해 회전량을 조절해 보세요.
- 웹앱: `left:30` / `right:75` 처럼 세기를 붙여 전송
- 펌웨어: `cmd.split(':')`로 파싱해 `motor.drive(dir, speed)`에 반영
- 이것이 실제 로봇의 **P 제어(비례 제어)** 입니다!

### 🥇 도전 C — 나만의 물체 학습 (YOLO 전이학습)
COCO에 없는 물체(학급 마스코트, 특정 장난감)를 탐지하도록 YOLO26을 전이학습 시켜 보세요.
- Roboflow(무료)로 사진 100장에 박스 라벨링 → YOLO 형식 내보내기
- 코랩: `model.train(data="my.yaml", epochs=50, imgsz=320)` → `export(format="onnx", imgsz=320)`
- 웹앱: `COCO` 배열을 내 클래스 이름으로 교체
- 분류 실습과 비교해 **라벨링 비용**이 얼마나 다른지 체감해 보세요 — 탐지 데이터가 비싼 이유!
